# Phase 2 — Digital Twin Core (complete + verified)

## Shipped
- **System-prompt assembler** (`lib/ai/system-prompt.ts`) — company-scoped; loads Michael Core ('all' + this company only) + recent decisions; stable hard-ceiling/human-only/injection preamble returned separately so it's prompt-cached. Zero cross-company references.
- **Confidence** (`lib/ai/confidence.ts`) — per-company-per-domain, earned from sample counts; dead-man check.
- **Twin pipeline** (`lib/ai/twin.ts`) — injection → human-only → confidence → hard ceilings → risk matrix → Opus (cached preamble) → output ceiling re-check → 60s queue OR escalate → audit → learn (increment sample). Escalated drafts persist to `send_queue` as `pending_review`.
- **APIs**: `/api/twin/respond` (anomaly-checked, role-filtered output), `/api/twin/approve` (Send as Michael), `/api/twin/escalate` (cancel), `/api/ea/queue`, `/api/people`.
- **UI**: EA dashboard (review queue + Send-as-Michael/Cancel, live stats) + seamless team chat (person selector, pending-state). Added to Command nav (role-gated).
- 8 placeholder people seeded.

## Verified end-to-end on the LIVE deployment
Posted a ₱800K pricing question as the VB GM → twin response:
- `status 200, tier 3, canAutoSend false, confidence 0`
- text = "Let me check on this and get back to you." (escalated, did NOT fabricate)
- `audit_log`: `bot / escalated / tier3 / auto=false` ✓
- `send_queue`: `pending_review tier3` draft ✓ (surfaces in EA dashboard, cron won't deliver it)
- `autonomy_domains`: `vb/pricing samples=1 conf=2%` ✓ (learning loop works)

Risk matrix behaved correctly: medium impact (₱800K) + 0% confidence → Michael-only. **Nothing auto-sent.**
